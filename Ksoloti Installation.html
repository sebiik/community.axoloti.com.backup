Ksoloti Installation


The install process of Ksoloti is identical to Axoloti BUT you need to install the 1.0.12-2 release and need to do a few extra steps.



Download the release package and runtime package for your OS here:

https://github.com/axoloti/axoloti/releases/tag/1.0.12-2/


The Axoloti community forum is probably down, so you can access the installation guide via the backup here:

https://sebiik.github.io/community.axoloti.com.backup/t/quick-start-guide/57.html


Before you connect Ksoloti, a few more steps are to be done:


1. Clone or download https://github.com/ksoloti/ksoloti and place it in a location of your choice.


2. Start the patcher, go to preferences and change the "firmware dir" to the subfolder "/firmware" of the location where you placed the above ksoloti folder.




3. Clone or download https://github.com/ksoloti/ksoloti-factory and place it in a location of your choice


4. Restart the patcher.


5. Back in the Patcher preferences, enter the location where you saved the ksoloti-factory folder in the field “Directory”. Add the git URL https://github.com/ksoloti/ksoloti-factory.git in the Remote Path to be able to update the library via the Patcher’s “Sync Libraries” function.



There is probably a simpler way by just selecting git and entering the URL, then Axoloti will download the library automatically, but this whole git business in the Patcher is pretty unintuitive…


The first time you check the “connect” checkmark for your Ksoloti, the patcher might give you a “firmware CRC mismatch” warning and ask if you want to update the firmware. Click "yes" (though you may need to click board > firmware > compile first)


Do not use the [board > firmware > flash] option because this assumes you want to revert your firmware to the stock Axoloti. The patcher will have the audacity to change the “Firmware Dir” back to stock Axoloti, save the preferences, and flash the wrong firmware to your board. You will have to set the Firmware Dir back as described in step 2. Flashing the stock firmware will do no harm, but will simply not work on Ksoloti. Try the [flash (user)] option if there is one in your menu, or simply restart the patcher and let it offer you to update the board firmware (“CRC mismatch”).


Currently there is no easy way to toggle between stock Axoloti boards and Ksoloti boards. Updating the patcher to recognize both, and at least be able to connect to either Axo or Kso, is planned.
