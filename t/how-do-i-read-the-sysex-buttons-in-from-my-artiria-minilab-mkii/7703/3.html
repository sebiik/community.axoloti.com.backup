janvantomme 2022-04-25 11:26:36 UTC #2

You'll need to write a custom object for that, since SysEx messages will vary depending on the manufacturer. There are some methods in the firmware that may be useful.

github.com

axoloti/axoloti/blob/46f6e4b383ce182da9dcca25b9d4b544fe20f990/firmware/midi_decoder.c

/*
 * midi_decoder.c
 *
 *      Author: jtaelman
 */

#include "ch.h"
#include "midi_decoder.h"


static const unsigned char StatusLengthLookup[16] = {0, 0, 0, 0, 0, 0, 0, 0, 3, // 0x80=note off, 3 bytes
                                               3, // 0x90=note on, 3 bytes
                                               3, // 0xa0=poly pressure, 3 bytes
                                               3, // 0xb0=control change, 3 bytes
                                               2, // 0xc0=program change, 2 bytes
                                               2, // 0xd0=channel pressure, 2 bytes
                                               3, // 0xe0=pitch bend, 3 bytes
                                               -1 // 0xf0=other things. may vary.
    };

Yeah, it looks like an object on its own can't implement SysEx reads, since the MIDI section doesn't have a signature that would support that. A separate MIDI SysEx section would be appropriate.
