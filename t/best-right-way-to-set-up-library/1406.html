have you watched this:

this is for 1.0.8 and above only, where libraries are introduced.

note: the video shows a private library which is synchronised to git, but if you just want it locally, then you simply need to create a library, with type 'local' and then select a directory for your library, and ensure you place 2 directories in it, one called patches, the other objects.

there is no 'patch browser', the object browser is for objects that are to be used within patches. (including sub patches (axs) ) , the intention is to introduce a patch browser, but as always finding time to do this is proving 'challenging'

(... in fairness, I think this is a common misconception that patches are displayed in the object browser.. not quite sure why, I think perhaps wishful thinking )
