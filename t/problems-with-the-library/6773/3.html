jaffasplaffa 2020-08-04 20:20:33 UTC #4

I think to use the com-lib it is mandatory to have a github account. But not 100% sure to be honest. I thought it was, but if you accessed it before with out, apparently it's not.

I just did a sync library and it seems to be working fine here.

Not sure where to go from here. Maybe @thetechnobear knows more or someone else that knows more about the Axoloti system.

For now, until you get this sorted out, you can still access the com-lib objects, by downloading them directly from Github and then create your own local library folder from the Axo editor.

You can download the com-lib manually here:

github.com

axoloti/axoloti-contrib

User contributed objects and patches for Axoloti. Contribute to axoloti/axoloti-contrib development by creating an account on GitHub.

I know, it's not a fix, but at least you can do some patching and have some fun while figuring the real problem out
