Challenger: @johannes

Challenge: noodles patches - self-playing improvisation patches

create a patch which plays on its own accord, and can be influenced/controlled by the user with 2 compositional controls, at top of patch, and clearly labelled

Winner: @philoop with sineseek

Submission window: Fri 29th 00:00 - Sun 31st Jan 23:59 (CET)

Voting window: Mon 1st - Fri 5th Feb 23:59 (CET)

Constraints:

  • maximum 2 parameters for real time composition control.
  • single patch file (axp)

Voting Criteria:

  • Awesomeness
  • More awesomeness
  • Patch clarity (is it still possible to see how the patch works?)

THIS THREAD IS FOR SUBMISSIONS ONLY
any comments/clarifications please use discussion thread.
you can submit as many patches as you wish, and update them before the voting windows opens.
please keep your submissions as one post per user
if you update a patch, please remove the previous version and place a date/time next to the update, so others know to try it again.

Reminder: patch must be self contained in one AXP file.

VOTING WILL BE ON THIS THREAD, it will be by a POLL created when the window opens.
you can try patches before voting begins, but please bare in mind patches may be updated.

Hope everyone has had fun patching, and theres still time if you haven't had time yet...
Im looking forward to trying the submissions

TIP:
you dont need to download patches to try them...simply
a) Right Click link in web browser, select 'Copy Link' (or whatever it is to copy links to clipboard in your browser)
b) In Axoloti, File menu - Open in URL , then paste the link , OK
c) hit LIVE

note: this does mean the 'link hit count' will not in increase on this website, so submitters don't assume the hit count is representative of the number of users trying, if they do the above, which is the easy' way it wont increase the count.

cpwitz 2016-01-31 13:15:59 UTC #5

Here's my humble submission: Automobeep.

Probability-driven random notes and drum hits, 4 voices (drums, bass, beeps and space glissando), simple progression and random notes from a pentatonic scale. With the "More" dial the probability of notes and therefore the produced mayhem can be increased.

automobeep.axp (89.8 KB)

There's one ugly thing with this patch: The probability-trigger subpatches are copy and paste, because you cannot re-use one internal subpatch. These should actually be external subpatches with the same code base.

I'm envious of patches like mtyas' because of their experimental coolness and playfulness. Me, I always stick to the common melodic and rhythmic patterns - but, hey
