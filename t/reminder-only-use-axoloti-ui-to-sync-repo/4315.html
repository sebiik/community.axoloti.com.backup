Reminder: only use Axoloti UI to sync repo

Community Library
thetechnobear 2018-03-05 13:27:04 UTC #1

Please can I remind users only to use the Axoloti UI to 'sync' changes to the community libraries.
(do not use native git commands/tools)

we have had a number of issues recently where users have been unable to sync, because another user has (inadvertently) changed their files. the axoloti UI prevents this happening ... but if some users side step this by using native git commands, then its very easy to screw this up if you don't take care/think it through.

I recognise for some more experienced users this is a limitation, but its vital that we do not create merge conflict situations. the majority of users have neither the experience/nor tools and setup to resolve them.
this is why I chose a very simple workflow/model - to keep it simple for all.

if you do are an experience developer with git experience, then sure you can do it, but only if you ensure:
- any change you commit ONLY affects your files - NO exceptions!
- you keep the operations simple, to ensure there can be no conflicts (e.g. i dont want to see history changes)

in the end , if you do stuff, and I don't get issues from other users, then I don't care too much...
Im only coming banging on your door if its F' up

please this is important, this potentially affects all users... and when it goes wrong, I end up spending quite a bit of time resolving the issues ... and frankly, Im getting a bit tired of it
