thetechnobear 2017-11-05 16:56:24 UTC #2

its certainly is not normal... if there users were seeing it as being unstable, there would (rightly ) be a huge number of complaints here...

but your gonna have to supply a bit more information than just that 'it crashes'...
what we need to determine is why your system is different from others, so we need to know exactly what you have setup, and how you are using it.

  • what do you mean by crashing? the editor? the patches?
  • what OS are you using?
  • have you installed the latest version of the Axoloti software?
  • are you using an SD card?
  • tried different usb cables?
  • are any errors shown in the axoloti console?

( you should be using the release version of 1.0.12)

also please, go an re-read the installation instructions and also the troubleshooting posts, in the user-guide section... perhaps this might given you an idea of where the issue may be.

without the above, hard to say whats going on, but given your saying its running for a few minutes...
one important thing for us to determine is, is the hardware crashing, or is it just disconnecting from the editor
(something many users , still attribute to it 'crashing', but its not really)

so try a few things...
create a simple patch which outputs a constant sine wave, keep this running... does it disconnect from the editor? if it does, do you still here the sine wave? (i.e. is the patch still running on the hardware, despite the editor disconnecting)

also try setting this as a startup patch, see if it works without the editor
