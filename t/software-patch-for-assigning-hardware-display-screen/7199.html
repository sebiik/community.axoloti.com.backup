There is a ton of info in this thread:

SPI/I2C OLED display

AFAIK, I2C multiplexers only work the other way around, i.e. to connect one master to several busses. As the multiplexer gets its switching commands via I2C, you can't just connect it "backwards". But any analog multiplexer (e.g. 4067) should be able to switch I2C. Also you could use a handshake signal and software to make sure that only one master at a time uses the display.
