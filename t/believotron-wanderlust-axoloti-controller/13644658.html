mongrol 2017-10-10 05:40:34 UTC #35

I assume you have some other microcontroller in there driving displays and possibly the controls as well. Can you elaborate on the design?

DazzlingDuke 2017-10-11 17:56:44 UTC #36

There isn't a microcontroller. Everything is handled by the Cortex M4 on the Axoloti core. The core is good for processing the ADCs in realtime. The OLEDs, though have to be limited. 128x32x4 is a lot of data to be updating at 20Hz, so it's not intended to be a fast display. It is, however pretty good at updating a few times a second without crunching the audio.

I would love to hear what sort of things you would want to display. If we can pull it off, we'll post pics.

If you're interested in hardware with a faster GUI update, we're working on a raspberry pi + axoloti core version early next year.

Here is a system diagram of the Wanderlust. Let me know if you have any questions.

Believotron Wanderlust System Diagram.png651x731 22.7 KB

You can find the open source schematic, PCB, and gerber files here: https://github.com/Believotron/BelievotronWanderlustHardware/tree/master/Beta1

The code is available on our development repo, but I swear you should wait until we set up a release repo, because that is a pile of sedimentary code and designs .

DazzlingDuke 2017-10-11 18:14:40 UTC #37

In other news, we had such a great response for our drum kit that we brought to the Chicago Full Moon Jam that we're going to be spending some time developing the kit further.

We're eagerly awaiting these two drum pad midi controllers, so we can get a good feel for how the software works with so many buttons.

We're trying this low-cost non-pressure sensitive drum pad because it has so many pads:

image.jpg1000x1000 174 KB

I've had issues with Novation knobs before, but this is all buttons, so it should integrate well.

The plan is to assign each row / column to be a set of 8 different variations on a drum sound. For example the first pad might be a kick drum with low resonance and each other button in the row increases the resonance.

This might also make a good step-sequencer, but it depends on how hard it is to implement the protocol to change the LEDs.

There are a lot of different ways to program this, so I'd be interested in hearing any suggestions for design metaphors.

The other controller we're working with is the Akai MPD218. It has a lot fewer pads, but they are pressure sensitive.
