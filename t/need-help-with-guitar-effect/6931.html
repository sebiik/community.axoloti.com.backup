patatos 2020-11-09 11:31:00 UTC #2

I don't quite understand what you want to achieve .. your patch looks like some sort of mono tap delay and then a chorus followed by a reverb and a compressor. However it has a really nice sound! I can give you some advice: if you want to use it as an insert effect you have to inject the clean signal somewhere after the delay block, also don't use the smooth object at 64 because it just kills every variation and the five dials connected to the rev are not needed, they are simply a copy of the controls already implemented.
And there is another issue: you declare three delay lines dl1, dl1_ and dl1__ , but actually you use only one of them: dl1.

patatos 2020-11-09 16:44:12 UTC #4

If you put a math / * c object before the mixer "bus in" you can adjust the clean signal .. The compressor is not a bad idea, but sadly it is not well adjusted, it is not your fault, it starts working for threshold <4 and you have to increase the ratio accordingly. You should also put the object drj / audio / out_stereo_vol to compensate for the gain loss of the compressor.
