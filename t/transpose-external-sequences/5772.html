ricard 2019-06-11 22:48:58 UTC #7

@altdelay, @reubenfinger: the transpose object (and thru object too for that matter) has a device parameter which specifies which device it operates on. In the example in the other thread, it is set to din, so it takes data from the MIDI IN DIN jack, processes (transposes) it, and send the result to the MIDI OUT DIN jack.

The midi/in/keyb object is just there to provide the drive for the red led, it's not actually used to read MIDI data for transposing.
