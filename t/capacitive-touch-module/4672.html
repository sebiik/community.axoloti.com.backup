deadsy 2018-06-21 04:59:03 UTC #7

I have this minor obsession with omnichord strum plates, so I'm interested in touch sensors.
Looking at the tp229 datasheet it appears to be a read-only i2c device. ie- read it periodically and get 16 bits representing the touch plate states. I think it should be possible to do the i2c access entirely within the patch object (ie- no drivers added to the main firmware)- that would be convenient.

The tricky thing is that you don't want to wait in the object for the device access to complete because i2c is slow.

I have an mpr121 device at the moment. One of these.

I'll probably just make that work first. The tp229 object would be very similar.
