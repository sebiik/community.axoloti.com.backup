thetechnobear 2018-02-10 11:25:09 UTC #2

this is caused by an old JRE...

this can be fixed as described here:

Axoloti patcher crashes to desktop on invoking file dialog

I just confirmed my suspicion - the self-contained Axoloti application includes an old version of the JRE - it is version 8 update 121, whereas the JRE installed on my system is update 151. (Verified by inspecting the properties of "C:\Program Files (x86)\Axoloti\runtime\bin\java.dll" and checking the version.) This explains why running Axoloti.exe results in the crash whereas running the .jar file directly does not. As a workaround, you can replace the "C:\Program Files (x86)\Axoloti\runtime"â€¦

background: the issue is, we bundled a copy of the JRE with the patcher, so that the user did not have to start installing java , and so we know everyone uses the same version (so has same behaviour) - unfortunately it appears one of the Windows 10 releases has broken this version. When the next version of axoloti is released , it will come with the newer version of the JRE and so resolve this issue. However, Im not sure when this is planned for.
