SirSickSik 2016-08-06 18:51:55 UTC #295

NEW MODULES

PATCH

"sendMatrix"
"receiveMatrix"
"dispMatrix"

the first is a module to write the controller value to a table or to get it's value to the controller when another destination/preset/output is being selected
Can be controlled by GPIOS controls using the CV input and button-inputs to walk through the menu

the receiveMatrix reads from the table and outputs 8 signals per module. A selector is used to select which instance it is. (this could even be manually controllable if you want, such as to switch control-values between groups of parameters)

the dispMatrix is a quick way to show which destinations are being selected.

NOTE though, that when two controls select the same destination, the destination will get the valuefrom the last module in the direction of the code. (eg most-right module)

SirSickSik 2016-08-08 10:45:18 UTC #299

NEW MODULES

NOISE

"noiseOsc"
noise generator that updates at the rate of the internal phase
a 12dbLP/12dBHP bandpass filter then filters the generated noise
both pitch and cutoff are controllable

MATH

"expCurveA" and "expCurveK"
audio rate and control rate stackable exponential curve generators build from a for-loop of ___SMMUL functions, with the exp parameter mixing between multiplying with a fixed value(no change) and multiplying with it self (curving) . The result will probably not be an exact exponential curve, but at least it's kinda cpu low and able to give a exponential-like curve to certain signals (envelopes for example)

"bipAmp"
bipolar attenuator of incoming signal
