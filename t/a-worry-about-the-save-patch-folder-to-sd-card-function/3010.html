A worry about the "save patch folder to SD-card" function

Software
jaffasplaffa 2017-03-05 09:53:11 UTC #1

Hello!

I was just looking through my sd-card for Axoloti, for cleaning. And this time I emptied my trash can, before moving all the automatically created patch folder to the trash, so I could count how many there was. There was more than 1000 empty patch folders! That is a lot. Way to much, IMO.

I am wondering if it would be possible to implement to be able to set a destination folder for these patch folders in preferences? So a user could just save them to a folder like "Sd-card/patches/". And then, when cleaning/maintaining, then just go into "Sd-card/patches/" and delete all the empty folders from there.

This would really be a lot safer.

I am very worried that I one day by mistake deletes a folder by mistake which actually contains something that I am using. It is very easy to delete a wrong folder when you have to delete more than 1000 unwanted path folders from the root of the sd-card. They are just messing up the sd-card and making it slower.

I think this is something that should be looked into, so I am bringing it up again. I am also curious about how other people use the patch folders, to get an overview how much they are actually used. To me they are just giving me worries, to be honest.

Thanks, Jaffa
