SmashedTransistors 2018-11-11 14:15:07 UTC #115

Hi, I added some updated OLED objects.

The new tiar/HW/OLED128x64nice is nice because it will be nicer to other hardware connected to the I2C. (hence with this one, the presence of an factory/i2c/config object is mandatory).
If you have two OLEDs connected with different I2C addresses, you can use two
OLED128x64nice objects.

modes:

  • mode 0: 4 lines of 11 characters (or "bars", see below).
  • mode 1: 1 line of 11 chars and oscilloscope.
  • mode 2: full height oscilloscope.

bars2.PNG996x429 36.5 KB

The tiar/HW/bar object connects to a line input of the OLED128x64nice object and make it draw an horizontal bar. If the tiar/HW/bar input is positive, the bar will be filled. If the input is negative, the displayed bar will be hollow.

The tiar/HW/bardbl object draws two bars.

The tiar/HW/strbar and tiar/HW/strbardbl writes a 5 character string before the bars:
